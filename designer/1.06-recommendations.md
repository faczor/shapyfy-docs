## 2. Human-Readable Summary
**User Behavior**
After answering the personalization questions, the user sees a loading screen while the app finds the best workout plans. Once loaded, the user is presented with 3 options: one "Recommended" plan that best fits their goals, and two alternatives. They can tap any plan to preview it.

**Key Cases**
*   **Loading**: Shows a spinner and "Finding your perfect plan..." text.
*   **Success**: Displays 3 plans. The top one is highlighted as "Recommended".
*   **Error**: If the network fails, an error message appears with a "Try Again" button and an option to build a workout manually.

**Data Displayed**
*   **Recommended Plan**: Name, description, "BEST FIT" badge, duration (days/week), number of exercises.
*   **Alternative Plans**: Name, description, duration, number of exercises.

## 4. Notes
*   **Design Philosophy**: Clear distinction between "Recommended" and "Alternatives". Recommended plan feels premium and tailored.
*   **Accessibility Features**:
    *   **Contrast**: All text meets WCAG AA/AAA.
    *   **Screen Reader**: Cards announced with full context ("Recommended...").
*   **Developer Notes**:
    *   **API**: Expects `recommended` object and `alternatives` list.
    *   **Timeout**: 5s timeout triggers error state.
    *   **Gradient**: Use `Brush.linearGradient` for the recommended card.
