## 2. Human-Readable Summary

**User Behavior**

The user manages the days of their workout plan using an **inline expandable card** pattern. Days are listed as collapsible cards ‚Äî tap to expand and see exercises, tap again to collapse. Users can add new days (Workout or Rest), reorder them via drag-and-drop, edit day names inline, and delete days. Each workout day contains a list of exercises with their target sets/reps/weight.

**The "Quick Add with Smart Defaults" Pattern:**
When adding exercises, the user taps "+ Add Exercise" which opens the Exercise Picker (2.01). They can add **multiple exercises in one trip** ‚Äî tapping an exercise **immediately adds it** with smart defaults (from history > AI > fallback). A Smart Toast confirms the addition and offers optional [Configure] to adjust targets. User taps "Done (X)" to return to the plan. This aligns with Brand DNA's "Zero Friction" and "Smart Defaults" principles.

**Key Cases**
*   **Empty State**: No days added yet ‚Äî prominent "Add your first day" CTA.
*   **Collapsed Day**: Shows day name, type badge, exercise count ‚Äî tap to expand.
*   **Expanded Day (Workout)**: Shows exercise list, "Add Exercise" button, delete day option.
*   **Expanded Day (Rest)**: Shows motivational message, delete day option.
*   **Rest Day**: No exercises needed ‚Äî peaceful recovery message.
*   **Workout Day**: At least 1 exercise recommended (can save empty for draft purposes).

**Data Displayed**
*   **Plan Header**: Plan name, total days count, total exercises count.
*   **Day Cards (Collapsed)**: Chevron, day number + name, type toggle, exercise count.
*   **Day Cards (Expanded)**: Editable name field, type toggle, exercise list or rest message.
*   **Exercise Rows** (inline): Drag handle, exercise name, category, targets (sets √ó reps @ weight), menu.

**Interaction & Motion**
Calm, premium feel aligned with Brand DNA. Day cards expand/collapse with spring physics (300ms, damping 0.8). Day reordering uses spring animations. Exercise rows support swipe-to-delete with elastic resistance. Type toggle uses liquid morph animation. All interactions have appropriate haptic feedback. **No bottom sheets for day editing ‚Äî everything happens inline.**

### Design Philosophy

**Inline Expandable Day Cards:**
- User maintains context ‚Äî sees entire plan while editing a day
- No navigation stack to manage ‚Äî tap to expand, tap to collapse
- Simpler mental model than bottom sheets or separate screens
- Immediate visual feedback when exercises are added

**"Stay in Picker" Pattern:**
- Eliminates tedious round-trips when adding multiple exercises
- User adds all exercises in one trip to Exercise Picker
- Counter shows progress ("3 added")
- "Save & Add Another" keeps flow going
- "Done (X)" provides clear exit point

**Auto-Save Everything:**
- Aligns with Brand DNA "Optimistic UI" ‚Äî changes save instantly
- No "Save Day" button ‚Äî reduces cognitive load
- "Finish & Save" at screen level confirms user is done with entire plan
- User never loses work, even on app crash

**Swipe-to-Delete:**
- Industry standard pattern (muscle memory)
- Undo toast prevents accidents
- More efficient than menu ‚Üí delete ‚Üí confirm

**Auto-Generated Day Names:**
- Reduces friction for quick setup
- User can always rename inline
- Pattern: "Day 1", "Day 2", etc.

**Rest Day Messaging:**
- "Recovery is part of progress" ‚Äî positive framing
- Educates users that rest is intentional, not lazy
- Calm, supportive tone per Brand DNA

### Brand DNA Compliance

- **"Soft Tech / Calm Focus"**: Spring animations, subtle glows, no jarring transitions
- **"Zero Friction"**: Swipe-to-delete, auto-generated names, single-tap actions
- **"Excel Killer"**: Data-dense exercise list but premium feel
- **"Optimistic UI"**: Instant feedback, undo instead of confirm-before-delete
- **"Digital Coach"**: Rest day messaging educates without lecturing

### Edge Cases Handled

1. **Max days limit**: Consider cap (e.g., 14 days) ‚Äî show toast if exceeded
2. **Duplicate day names**: Allowed (user may want "Push", "Push", "Pull")
3. **Very long day name**: Truncate with ellipsis, max 30 characters
4. **Many exercises (scrolling)**: Expanded card grows, screen scrolls
5. **Rapid swipe-delete**: Debounce undo toasts (only latest shows)
6. **Offline mode**: All actions work locally, sync when online
7. **Switch type mid-edit**: Confirmation prevents accidental exercise loss
8. **Adding Mode interrupted**: If user backs out of Exercise Picker, exercises already added remain saved
9. **Multiple days expanded**: Only one day can be expanded at a time ‚Äî tapping another collapses the first
