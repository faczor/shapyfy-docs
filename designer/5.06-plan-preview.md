## 2. Human-Readable Summary

**User Behavior**
The user views a workout plan's full details before activating it or starting a workout. This is a **unified screen** that adapts based on contextâ€”whether the user came from onboarding recommendations, is viewing their active plan, or browsing an inactive saved plan.

**Key Cases**
*   **Onboarding Context**: User sees recommended plan for the first time. Two actions: "Activate Plan" (start using it) or "Customize First" (edit before activating).
*   **Active Plan Context**: User is already following this plan. Shows progress (week/day, progress bar). One action: "Start Day X" to begin today's workout.
*   **Inactive Plan Context**: User is viewing a saved plan they're not currently following. One action: "Activate Plan" to switch to this plan.
*   **Day Expansion**: User expands any day card to see exercise list. Can tap expanded list to view detailed workout preview (5.05).
*   **Edit Flow**: User taps [â‹®] menu â†’ "Edit Plan" â†’ navigates to Plan Builder (5.03) for full editing.

**Data Displayed**
*   **Hero Card**: Plan name, metadata (duration, total exercises), context-specific info (progress bar OR "last used").
*   **Day Cards**: Day number, day name, exercise count. When expanded: full exercise list with targets.
*   **Progress Indicators** (Active only): Checkmarks for completed days, play icon for current day.

**Interaction & Motion**
The screen feels **calm and informational**. Day cards expand smoothly to reveal exercises. The Hero Card is the anchorâ€”always visible at top, providing context and access to secondary actions via the [â‹®] menu. Bottom action bar provides the primary CTA, keeping critical actions in the **thumb zone**.
