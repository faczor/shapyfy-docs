## 2. Human-Readable Summary
**User Behavior**
The user has just tapped "Let's Log Your First Set" and is now presented with a choice of exercises to try. They see three large, premium-looking cards representing:
*   **List**: 3 selectable exercises (Bench Press, Dumbbell Press, Squat)

**Interaction & Motion**
Exercises slide in with a staggered animation, making the list feel dynamic. Selecting an exercise triggers a smooth expansion effect, transitioning seamlessly to the detail view.


**Key Cases**
*   **Standard Flow**: User sees cards drop in with a staggered animation and selects one.
*   **Reduced Motion**: Animations are skipped.
*   **Small Screens**: The list is scrollable if all cards don't fit.

**Data Displayed**
*   **Title**: "Which one excites you?"
*   **Exercise Cards**:
    *   **Bench Press**: Chest ¬∑ Barbell, "Classic upper body builder"
    *   **Dumbbell Press**: Chest ¬∑ Dumbbell, "Build balanced strength"
    *   **Squat**: Legs ¬∑ Barbell, "Legendary leg developer"

**Card 1 (Bench Press):**
- Initial state: TranslateY -200dp (above screen), Opacity 0
- Animates to: TranslateY 0, Opacity 1
- Duration: 500ms
- Easing: **Spring animation** with bounce
  - Spring damping ratio: **0.7f** (refined from 0.6f - subtler, more premium feel)
  - Spring stiffness: `Spring.StiffnessMedium`
- Delay: 200ms
- **Effect:** Card drops in from above and bounces once when landing

**Card 2 (Dumbbell Press):**
- Same animation as Card 1
- Delay: 300ms (100ms after Card 1)

**Card 3 (Squat):**
- Same animation as Card 1
- Delay: 400ms (100ms after Card 2)

**Purpose:** Creates "platforms landing" effect. Each card feels like a solid platform dropping into place, inviting user to choose.

#### Card Interaction States

**Default State (No Interaction)**

**Pressed State (User Actively Pressing Card)**

**Visual changes to PRESSED card:**

**Other cards (not pressed):**
- No visual changes
- Remain in default state

**Haptic feedback:**
- Type: Medium impact
- Timing: On press down (not release)

**Release Behavior**
- Card returns to default state: Elevation 16dp ‚Üí 12dp
- Duration: 100ms
- Easing: Ease-out
- **Navigation:** Instant transition to Exercise Detail Screen (1.03)
  - No loading state needed

#### Return from Exercise Detail (Back Navigation)

**When:** User taps back arrow from Exercise Detail Screen and returns here

**Behavior:**
- Cards remain in their current state (no re-animation)
- Screen appears instantly (no entrance animations)
- All 3 cards visible and tappable

**Purpose:** Keeps interaction fast. Re-animating cards on return would feel slow and unnecessary.

#### Design Rationale

**Why this achieves "Premium but Calmer" energy (v2 refinements):**

1. ‚úÖ **Refined elevation** (12dp) - Premium feel through precision, not maximum effects
2. ‚úÖ **Large size** (140dp min) - Each choice feels substantial, supports dynamic type
3. ‚úÖ **Strengthened gradient** - Lime‚Üídarker lime maintains proper 6-7/10 energy level
4. ‚úÖ **Clean white cards** - Creates visual breathing room, cleaner 1dp borders
5. ‚úÖ **Optimized animations** - Subtler bounce (0.7f damping), reduced-motion support
6. ‚úÖ **Accessible typography** - Better hierarchy (20sp title, 24sp names, 14sp prompts), WCAG AA compliance
7. ‚úÖ **Tactile interaction** - Simpler elevation-only feedback, more responsive
8. ‚úÖ **Personality differentiation** - Icon gradients help distinguish exercise types
9. ‚úÖ **Design system compliance** - All colors in Color.kt, standardized 16dp spacing
