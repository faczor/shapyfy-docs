## 2. Human-Readable Summary
**User Behavior**
If a user saves a workout program but doesn't start it immediately, a "Program Ready" banner appears at the top of their dashboard. This banner reminds them of their saved program and gives them a quick way to start Day 1 or view the full plan.

**Key Cases**
*   **Banner Visible**: When a program is saved as a draft and no active workout is in progress.
*   **Starting**: Tapping "Start Day 1" launches the workout.
*   **Viewing**: Tapping "View" opens the plan preview screen.

**Data Displayed**
*   **Status**: "Program Ready"
*   **Program Name**: e.g., "Upper/Lower Split"
*   **Next Workout**: e.g., "Next: Day 1 - Upper Body Â· 5 exercises"

## 4. Notes
*   **Design Philosophy**: Prominent but friendly reminder. Uses the "success" green color palette to encourage action.
*   **Developer Notes**:
    *   **Display Logic**: Show only if `status == DRAFT` and no active workout.
    *   **Persistence**: Persist across app restarts.
    *   **Dismissal**: Automatically hides when program starts or is replaced.
