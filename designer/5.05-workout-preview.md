## 2. Human-Readable Summary

**User Behavior**
The user reviews the detailed exercise list for a single workout day before starting it. This is the "last look" confirmation screen. The user sees all exercises, sets, reps, and target weights in a clean, scannable format. They can then tap "Start Workout" to begin logging.

**Key Cases**
*   **Standard View**: User sees 3-7 exercises with clear targets.
*   **Long Workout**: User sees 8+ exercises (scrollable).
*   **Single Exercise**: User sees 1 exercise (rare, but supported).
*   **Bodyweight Exercises**: User sees exercises without weight targets.

**Data Displayed**
*   **Day Context**: Day number and name (e.g., "Day 1: Upper Body").
*   **Workout Summary**: Total exercises and estimated duration.
*   **Exercise List**: Each exercise shows:
    *   Exercise name
    *   Muscle group category
    *   Target sets, reps, and weight
*   **Primary Action**: "Start Workout" button.

**Interaction & Motion**
This screen follows Shapyfy's "Soft Tech / Calm Focus" brand DNA. The design is calm and precise (coaching tone), not hyped or gamified. Screen entry is a simple 200ms fade-in with no stagger or pulse effects. The single "Start Workout" button provides zero-friction progression to the Active Workout screen.

**Muscle Group Icon:**
- Icon: Material Symbol `fitness_center` (Rounded, weight 500)

- Text: " · " (U+00B7 middle dot with spaces)

- Text: "[Sets]×[Reps] @ [Weight]kg"

**Format Rules:**
- Sets: Always show number (required field)
- Reps: Show range if min ≠ max (e.g., "8-12"), single if equal (e.g., "10")
- Weight: Show value + "kg" (or "lb" based on user settings), or "Bodyweight" if null
- Symbol: Use "×" (U+00D7 multiplication sign) not "x" (letter)
- Symbol: Use "@" for weight indicator

- This is a significant action (starting workout session)

### <workout_preview_animations>

#### Screen Entry Animation

**Philosophy:**
- Follows Brand DNA "Optimistic UI" and "Zero Friction"
- Simple, fast, no visual noise
- Dark mode is priority (avoid harsh transitions)

**Animation Sequence:**

**Phase 1: Full Screen Fade (0-200ms)**
- Entire screen fades in: Opacity 0 → 1
- Duration: 200ms
- Easing: LinearEasing (no complex curves)
- All elements appear together (no stagger)

**Why no stagger:**
- Stagger implies "loading" (violates "Optimistic UI")
- User is ready to review, not watch a show
- Follows "Calm Focus" (no unnecessary motion)

**Accessibility:**
- If "Reduce Motion" enabled: Instant appearance (no fade)
- System setting: `UIAccessibility.isReduceMotionEnabled` (iOS) / `Settings.Global.TRANSITION_ANIMATION_SCALE` (Android)

#### Button Press Feedback

**Phase 1: Press Down (0-100ms)**
- Scale: 1.0 → 0.98
- Background: #A8E6CF → #92D6B8
- Duration: 100ms
- Easing: LinearEasing
- Haptic: Medium impact (fires at 0ms)

**Phase 2: Release (100-200ms)**
- Scale: 0.98 → 1.0
- Duration: 100ms
- Easing: LinearEasing
- Navigation triggers at 150ms (mid-animation)

**Why trigger navigation mid-animation:**
- User feels instant response
- Avoids "wait for animation to finish" friction
- Follows "Optimistic UI" principle

### <workout_preview_edge_cases>

#### Long Exercise Names

**Scenario:** Exercise name > 30 characters (e.g., "Barbell Romanian Deadlift with 3-Second Eccentric")

**Behavior:**
- Truncate at 30 characters with ellipsis
- Display: "Barbell Romanian Deadli..."
- Full name: Available in Active Workout detail view (2.03)

**Layout:**
```
1  Barbell Romanian Deadli...
   Legs · 3×8 @ 80kg
```

**Why 30 characters:**
- Fits most exercise names without truncation
- Maintains visual rhythm (not jagged lines)
- Rare case (95% of exercise names < 25 chars)

#### Bodyweight Exercises

**Scenario:** Exercise has no target weight (e.g., Pull-Ups, Push-Ups, Dips)

**Behavior:**
- Show "Bodyweight" instead of weight value
- Display: "3×10 @ Bodyweight"

**Layout:**
```
1  Pull-Ups
   Back · 3×10 @ Bodyweight
```

**Why "Bodyweight" not "BW":**
- Clear and unambiguous (new users understand)
- "BW" is jargon (violates "Newbie" audience principle)
- Doesn't add significant visual clutter

#### Many Exercises (8+ exercises)

**Scenario:** Workout has 8+ exercises (scrollable content)

**Behavior:**
- Content area is scrollable (vertical)
- Bottom button remains fixed (always visible)
- No "5 of 10" pagination indicator (clutter)
- Scroll indicator: System default (iOS/Android)

**Why no pagination:**
- User can see all exercises by scrolling (intuitive)
- Pagination implies "pages" (wrong mental model)
- Follows "Calm Focus" (no extra UI elements)

#### Single Exercise

**Scenario:** Workout has only 1 exercise (rare but possible)

**Behavior:**
- Same layout (consistency)
- No dividers (only 1 item)
- Hero header shows "1 exercise · Est. 5 min"

**Layout:**
```
┌─────────────────────────────────────┐
│  Today's Workout                     │
│  1 exercise · Est. 5 min             │
│                                      │
│  ┌─────────────────────────────┐   │
│  │  1  Bench Press              │   │
│  │     Chest · 3×8-12 @ 80kg    │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

#### No Estimated Duration Data

**Scenario:** Backend doesn't provide estimated duration (edge case)

**Behavior:**
- Omit duration from metadata
- Display: "5 exercises" (no " · Est. X min")

**Why omit instead of "Unknown":**
- "Unknown" adds no value (states the obvious)
- Clean metadata (no clutter)
- Follows "Excel Killer" (show data or nothing)

#### Long Workout (90+ minutes)

**Scenario:** Workout estimated duration > 90 minutes

**Behavior:**
- Display hours + minutes: "Est. 1h 30min"
- Format: "[H]h [M]min" (no "hour" word for compactness)

**Example:**
- 45 min → "Est. 45 min"
- 90 min → "Est. 1h 30min"
- 120 min → "Est. 2h"

</workout_preview_edge_cases>

## 4. Notes

*   **Design Philosophy**: "Soft Tech / Calm Focus" (Brand DNA). This screen is a coaching moment, not a hype moment. User reviews the plan with their digital coach before starting. Clean, data-dense, precise.

*   **Brand Archetype**: "The Digital Coach" - Smart, Precise, Encouraging. Not hyped, not gamified, not childish.

*   **Differentiation from Plan Preview (1.07)**:
    *   Plan Preview (1.07) = Multi-day program overview ("Which plan do I want?")
    *   Workout Preview (5.05) = Single day detail view ("What's in today's workout?")

*   **Accessibility Features**:
    *   **Contrast**: All text meets WCAG AA minimum (4.5:1). Exercise names exceed AAA (7:1+).
    *   **Touch Targets**: All items ≥ 64dp height (exceeds 48dp minimum).
    *   **Screen Reader**: Clear hierarchy (heading "Today's Workout", list of exercises).
    *   **Reduce Motion**: No animation if system setting enabled.
    *   **Dynamic Type**: Text scales with user's font size preference.

*   **Developer Notes**:
    *   **Data Source**: `WorkoutDay` object from plan (contains exercise list, day name).
    *   **Duration Calculation**: Sum of (sets × rest time) + setup time (estimate 2 min per exercise).
    *   **Bodyweight Detection**: If `targetWeight == null`, display "Bodyweight".
    *   **Navigation**: Pass `planId`, `dayIndex`, and `exerciseList` to Active Workout (2.02).
    *   **No Loading State**: This screen assumes data is already loaded (from Dashboard or Plan Preview). If data is missing, show error state (not loading spinner).

*   **Future Enhancements (Post-MVP)**:
    *   **Expandable Detail**: Tap exercise → Inline expansion showing set-by-set breakdown.
    *   **Last Performance Badge**: Show "Last time: 82.5kg × 10 reps" below targets (if history exists).
    *   **Estimated Volume**: Show total volume (sets × reps × weight) in hero header.
    *   **Swipe to Skip**: Swipe exercise left → "Skip this exercise" option (if user doesn't have equipment).
